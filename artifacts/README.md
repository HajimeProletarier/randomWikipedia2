# Artifacts - 成果物管理

このディレクトリは、プロジェクトの要件定義書や設計書などのドキュメント成果物を管理します。

## ディレクトリ構成

```
artifacts/
├── README.md                    # このファイル
├── requirements-template.md     # 要件定義書テンプレート
└── [プロジェクト名]/            # 各プロジェクトごとのドキュメント
    ├── requirements.md          # 要件定義書
    ├── design.md                # 設計書
    └── ...
```

## ドキュメント作成の原則

### シンプルさを保つ
- 必要最小限の情報から始める
- YAGNI - 使わない情報は書かない
- 読む人を意識して書く

### 生きたドキュメント
- コードと同期させる
- 古くなったら更新または削除
- 「あとで更新」は「永遠に更新しない」

### テンプレートの使い方
1. `requirements-template.md` をコピー
2. プロジェクトディレクトリを作成
3. 必要なセクションだけ埋める
4. 不要なセクションは削除する
