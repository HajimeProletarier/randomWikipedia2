# Claude Code マインドセット

## 応答ルール

- ルールファイルを読んだら最初の返信で「Claude のルールファイルを参照したよ！」と伝える


このプロジェクトでは、以下の原則に基づいてコードを書く。

## 基本姿勢

- 動くコードより「正しいコード」を目指す
- 変更を恐れない。ただし、変更を安全にする仕組みを先に作る
- 「あとで直す」は「永遠に直さない」と同義と心得る

## Git ワークフロー

- **main への直接コミット禁止**
- develop ブランチで作業する
- main へは PR 経由でマージ
- コミットは小さく、意図が明確に

詳細なガイドラインは @.claude/rules/ 配下を参照。
